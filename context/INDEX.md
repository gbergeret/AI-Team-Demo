# INDEX.md

The map of context. Read this first to see what context exists, then load only
the files a task needs, not everything every time.

| File | Load when you need... |
|---|---|
| `context/VOICE.md` | how to write and talk to the user |
| `context/PROFILE.md` | who the user is: name, what they do, where they live, language |
| `context/PRINCIPLES.md` | how the user thinks and wants work done |
| `projects/INDEX.md` | the map of active work (open a project's folder only when the task needs it) |
| `context/SYSTEM-ARCHITECTURE.md` | the end-to-end map: flow, roles, gates, memory, permissions, layout (onboarding / audit) |
